# คู่มือการใช้งานระบบ (ฉบับส่งอาจารย์)
## โครงการ DUDUANG / FLORDER

## 1) บทนำ
เอกสารฉบับนี้จัดทำขึ้นเพื่ออธิบายการใช้งานระบบ DUDUANG ซึ่งเป็นระบบเว็บแอปพลิเคชันสำหรับการดูดวงร่วมกับการแนะนำสินค้าและการสั่งซื้อสินค้าออนไลน์แบบครบวงจร โดยมีการแยกบทบาทผู้ใช้งานเป็นผู้ใช้งานทั่วไป (User) และผู้ดูแลระบบ (Admin)

---

## 2) วัตถุประสงค์ของระบบ
1. เพื่อพัฒนาระบบ Full-Stack ที่เชื่อมโยงประสบการณ์ดูดวงกับการซื้อสินค้าในแพลตฟอร์มเดียว
2. เพื่อให้ผู้ใช้งานสามารถทำธุรกรรมได้ครบตั้งแต่การเลือกสินค้า สร้างออเดอร์ ชำระเงิน และติดตามสถานะ
3. เพื่อให้ผู้ดูแลระบบสามารถตรวจสอบสลิป ช่วยดำเนินงานออเดอร์ และจัดการข้อมูลสินค้าได้สะดวก
4. เพื่อประยุกต์ใช้แนวคิด Software Engineering ในการออกแบบสถาปัตยกรรม การยืนยันตัวตน และการจัดการข้อมูล

---

## 3) ขอบเขตการทำงานของระบบ
### 3.1 ขอบเขตฝั่งผู้ใช้งาน (User)
- สมัครสมาชิก / เข้าสู่ระบบ / ออกจากระบบ
- ดูดวงแบบ Interactive: ตอบคำถาม → เลือกไพ่ → รับคำทำนาย
- รับสินค้าแนะนำตามผลคำทำนาย
- เรียกดูรายละเอียดสินค้า, เพิ่มลงตะกร้า, สร้างคำสั่งซื้อ
- ชำระเงินด้วย PromptPay หรือ Credit Card (Mock)
- อัปโหลดสลิปสำหรับ PromptPay
- ติดตามคำสั่งซื้อและประวัติดูดวงย้อนหลัง
- แก้ไขโปรไฟล์และเปลี่ยนข้อมูลสำคัญ (อีเมล/รหัสผ่าน พร้อมยืนยันก่อนบันทึก)

### 3.2 ขอบเขตฝั่งผู้ดูแลระบบ (Admin)
- เข้าถึงหน้าแดชบอร์ดหลังบ้าน
- ดูภาพรวมคำสั่งซื้อและรายได้
- ตรวจสอบสลิปที่รออนุมัติ
- อัปเดตสถานะคำสั่งซื้อใน workflow ที่กำหนด
- จัดการสินค้า: เพิ่ม / แก้ไข / ลบ (soft delete) / เปิด-ปิดการขาย / ปรับสต็อก

---

## 4) ข้อจำกัดของระบบ
1. การชำระแบบ Credit Card เป็นการจำลองธุรกรรม (Mock) ยังไม่เชื่อม Payment Gateway จริง
2. การชำระแบบ PromptPay ใช้การอัปโหลดสลิปเพื่อให้แอดมินตรวจสอบแบบ manual
3. ระบบอ้างอิงฐานข้อมูลภายในโครงการ (PostgreSQL + Prisma) และไม่ได้เชื่อมระบบภายนอกด้านขนส่ง
4. การแจ้งเตือนแบบเรียลไทม์ (เช่น WebSocket/Push Notification) ยังไม่ได้เปิดใช้งาน
5. การจัดการสิทธิ์เป็น Role-Based แบบพื้นฐาน (User/Admin)

---

## 5) ขั้นตอนการใช้งานระบบ
### 5.1 ขั้นตอนใช้งานสำหรับ User
1. เข้าเว็บที่ `http://localhost:3000`
2. สมัครสมาชิกหรือเข้าสู่ระบบ
3. ไปหน้า `ดูดวง` และทำขั้นตอนจนได้ผลคำทำนาย
4. เลือกสินค้าแนะนำหรือเข้าสู่หน้าร้าน (`/shop`)
5. เพิ่มสินค้าลงตะกร้า (`/cart`) และยืนยันคำสั่งซื้อ
6. ไปหน้า `ชำระเงิน` (`/payment`)
   - PromptPay: สแกน QR และอัปโหลดสลิป
   - Credit Card: กรอกเลขบัตรครบ 16 หลักแล้วชำระ
7. ตรวจสอบผลลัพธ์ที่หน้า `pay-success` หรือ `pay-fail`
8. ติดตามคำสั่งซื้อที่ `/orders` และดูประวัติดูดวงที่ `/fate-history`

### 5.2 ขั้นตอนใช้งานสำหรับ Admin
1. เข้าสู่ระบบด้วยบัญชีแอดมิน
2. ระบบพาไปหน้า `/admin`
3. ตรวจรายการสลิปรออนุมัติ
4. อัปเดตสถานะคำสั่งซื้อให้ถูกต้องตาม workflow
5. จัดการสินค้าใน `/admin/products`

---

## 6) เงื่อนไขการใช้งานและความถูกต้องของข้อมูล
1. ผู้ใช้งานต้องเข้าสู่ระบบก่อนเข้าหน้าที่ต้องใช้สิทธิ์ (เช่น payment, profile, orders)
2. การเปลี่ยนอีเมล/รหัสผ่านต้องผ่านการตรวจสอบความถูกต้องก่อนบันทึก
3. ระบบตรวจสอบสินค้าคงคลังก่อนสร้างออเดอร์
4. เฉพาะแอดมินเท่านั้นที่เข้าถึง endpoint หลังบ้านและจัดการสินค้าได้

---

## 7) การทดสอบก่อนส่งงาน (Pre-submission Checklist)
1. สมัครสมาชิกและเข้าสู่ระบบได้
2. ดูดวงครบ flow และได้ผลคำทำนาย
3. เพิ่มสินค้าเข้าตะกร้าและสร้างออเดอร์ได้
4. PromptPay: อัปโหลดสลิปได้ และแอดมินอนุมัติได้
5. Credit Card: กรอกเลขบัตรครบ 16 หลักและชำระได้
6. ผู้ใช้ดูประวัติคำสั่งซื้อ/ประวัติดูดวงได้
7. แอดมินจัดการสินค้าได้ครบทุกฟังก์ชัน

---

## 8) สรุป
ระบบ DUDUANG รองรับกระบวนการใช้งานหลักครบทั้งฝั่ง User และ Admin ตั้งแต่การดูดวงจนถึงการสั่งซื้อและติดตามคำสั่งซื้อ โดยมีโครงสร้างที่พร้อมต่อยอดไปสู่ระบบ production ในขั้นถัดไป
